# Implementation & Constructive Problems

## 🎯 What to Expect
These are ad-hoc problems that test your ability to **translate problem statements into code** accurately. No specific algorithm — just clean implementation.

## 🧩 Key Patterns
- **Simulation**: Follow the process described step by step
- **Greedy Construction**: Build answer greedily based on observations
- **Edge case handling**: Watch for overflows, empty inputs, boundary conditions
- **Modular code**: Break logic into helper functions

## 💡 Tips
1. Read the problem statement **very carefully** — edge cases hide in constraints
2. Start with brute force, then optimize
3. Watch out for integer overflow — use `long long` in C++ / `long` in Java
4. Test with small inputs by hand before coding

## 📚 Common Interview Problems
- Rotate matrix 90 degrees
- Spiral order traversal
- Set matrix zeros
- Pascal's triangle
- Next permutation
